# Daniel Mall - 原始需求文档

> 面向电商客服的「工具订阅型产品」从 0 → 1 设计规范

---

## 一、产品本质

> **面向电商客服 / 客服主管 / 客服经理的「工具订阅型产品」**

| 阶段     | 策略                      |
| -------- | ------------------------- |
| 当前阶段 | 所有工具免费              |
| 未来阶段 | 工具订阅制 / 高级功能收费 |

**核心定位：**

- 「职业成长」只是工具中心中的一个子模块
- 产品核心价值：**解决客服真实工作问题，而不是教课**

---

## 二、目标用户与使用场景

### 2.1 目标用户

| 角色                | 说明         |
| ------------------- | ------------ |
| 电商客服            | 售前 / 售后  |
| 客服组长            | 小团队管理   |
| 客服主管            | 10-30 人团队 |
| 客服经理 / 服务运营 | 战略规划     |

### 2.2 高频使用场景

- 日常客服工作提效
- 客诉 / 投诉处理
- KPI 计算与复盘
- 管理汇报与总结
- SOP / 话术 / 文档快速复用

---

## 三、基础产品约束

### 3.1 登录与身份体系

**核心机制：匿名优先**

```
用户访问 → Splash 页面 → 判断是否已绑定邮箱
                              ├─ 已绑定 → 直接进入首页
                              └─ 未绑定 → 提示：绑定邮箱 / 跳过（继续匿名）
```

**关键要求：**

- 默认匿名使用（anonymous_id）
- 匿名数据在绑定邮箱后**必须合并**

### 3.2 前端技术约束

| 约束项        | 要求                                    |
| ------------- | --------------------------------------- |
| **响应式**    | PC 和移动端都有设计稿，双端高体验响应式 |
| **基准字号**  | `html { font-size: 16px; }` 固定不变    |
| **主题系统**  | CSS 变量 + TailwindCSS 设计系统         |
| **主题类型**  | 暗 / 亮 / Hero 三种主题，默认 Hero      |
| **Hero 主题** | 参考英雄联盟客户端高贵主题              |
| **动画规范**  | 阴影动画、变换、显隐需保证不突兀        |

## 八、工程与项目结构

### 8.1 技术栈

| 层级     | 技术选型                                       |
| -------- | ---------------------------------------------- |
| **前端** | Vue3 + TypeScript + SCSS + TailwindCSS + Nuxt4 |
| **后端** | Java 21 + Spring Boot + MySQL                  |

### 8.2 项目结构

```
daniel-mall/
├── ku-utils/        # 通用函数库（npm 包）
├── kv-ui/           # 自封装 Vue UI 组件库（npm 包）
├── daniel-web/      # 前端 Web 项目
├── domain-cms/      # 领域模型（用户 / 工具 / 订阅）
├── domain-server/   # 后端服务
└── docs/            # 产品 & 技术文档
```

## 九、文档生成顺序

按以下顺序生成设计文档：

| 步骤   | 文档                | 内容                                             |
| ------ | ------------------- | ------------------------------------------------ |
| Step 1 | 产品需求文档（PRD） | 产品定位、用户画像、使用流程、工具分类、MVP 范围 |
| Step 2 | 系统架构设计        | 前后端职责、工具引擎、数据流与扩展点             |
| Step 3 | 数据库设计          | 用户、工具、使用记录、收藏/历史                  |
| Step 4 | API 设计            | 工具执行、用户状态、历史记录 API                 |
| Step 5 | 前端结构设计        | 路由、页面划分、组件抽象                         |

## 十、设计原则

| 原则         | 说明                 |
| ------------ | -------------------- |
| **工具优先** | 工具优先于内容       |
| **简单直接** | 简单、直接、可复制   |
| **可扩展**   | 为订阅制预留结构     |
| **场景驱动** | 一切围绕客服真实工作 |

## 十一、最终目标

打造一个：

- ✅ **客服每天都会打开的工具箱**
- ✅ **可从免费自然过渡到订阅制**
- ✅ **能覆盖客服 / 主管 / 经理多个角色的 Web 产品**
